Competitors
Battle cards — how to position Planhat against each competitor
Enterprise trust note: SFDC / Microsoft / ServiceNow dominate enterprise. "SFDC is too costly" is not a credible argument from Planhat — you must earn trust via logos, security credentials, and integrations first. As one Lumen exec put it: it's easier to approve $10M more with SFDC than $100k with an unknown vendor.
CompetitorCategoryPositioningKey weakness vs Planhat
Gainsight CS Platform Acknowledge market leader status — then pivot to speed and flexibility Slower implementation, higher admin burden, less flexible AI. Planhat: native AI with BYO LLM (Claude, OpenAI, Gemini)
ChurnZero CS Platform Engagement automation only — no modular logic Lacks the modular workflow logic Planhat has. No multi-vendor data model
Totango CS Platform Predefined success blocks limit flexibility Success blocks are rigid — Planhat's custom objects and workflows are fully configurable
Vitally CS Platform Strong UX, weak data model Weak time-series data and poor parent/child hierarchy — critical for MSPs managing multi-vendor accounts
Salesforce CRM Linear sales tool — post-sale = expensive add-ons No time-series data natively. Post-sale requires Service Cloud + custom dev. Planhat is built for the full lifecycle out of the box
HubSpot CRM Not built for post-sale complexity Works for SMB sales teams. Falls apart at scale post-sale — no health scoring, no lifecycle automation
ConnectWise PSA Complement, don't replace PSA / ticketing only. Position Planhat as the CS layer on top via API integration — not a rip and replace
Ray Allen / ScalePad Asset Mgmt Asset lifecycle only No CS capabilities. Planhat ingests their asset data and adds health scoring, renewal automation, and expansion playbooks on top

POC Option 1 — Demo data
Immediate access
No cost. Limited user access. Supported by AE and TDS. Best for early-stage evaluation.
POC Option 2 — Sample data (NDA)
Customer's own data
Requires NDA. Takes 2–4 weeks to build proof environment. Best for mid-stage validation.
POC Option 3 — Enterprise POC (paid)
Full paid engagement
~3 months. Used for complex enterprise deals where full feasibility must be proven before contract.

Objection Handling
Responses by category — click to expand
Time / Attention▾
"This isn't a good time." / "Can you call me later?" / "Send me an email." / "About to jump into a meeting."
"Give me 30 seconds so I can explain the reasons for my call?" / "Can I call you back this afternoon?"
No Interest▾
"We're not interested."
"No probs. Can I just ask why — CS tool already in place? Not the right person? Bad time?"
"This isn't a priority right now."
"No probs. Where's your priority right now?" / "What would be a good time to reconnect?"
"I don't think we need this."
Probe into what they're using, what gaps exist in their current setup. Surface a pain they haven't articulated.
Already Have a Tool — Gainsight▾
"We use Gainsight."
Acknowledge their choice. Then: "Gainsight is a strong platform — a lot of teams come to us because of the implementation complexity and admin overhead. How long did it take to get fully live?" (Surface time-to-value and flexibility pain.) Key differentiators: faster implementation, lower admin burden, native AI with BYO LLM (Claude, OpenAI, Gemini).
Already Have a Tool — ChurnZero▾
"We use ChurnZero."
ChurnZero is built for engagement automation but lacks modular workflow logic and a multi-vendor data model. Ask: "How do you handle renewal tracking across multiple vendor lines today — is that all in ChurnZero or are you pulling from other places too?"
Already Have a Tool — Totango / Vitally▾
"We use Totango." / "We use Vitally."
Totango: Predefined success blocks limit flexibility. Planhat's custom objects and workflows are fully configurable for any vendor lifecycle model.

Vitally: Strong UX but weak time-series data and poor parent/child hierarchy — critical issues for MSPs managing multi-vendor portfolios at scale.
Already Have a Tool — Salesforce▾
"We do this in Salesforce." / "We already have Salesforce."
SFDC is a linear sales tool — post-sale needs (health scoring, lifecycle automation, time-series data) require expensive add-ons and significant customization. Don't argue against SFDC directly. Instead: "Totally — most of our customers still use SFDC for their sales CRM and run Planhat alongside it for the post-sale motion. We have a native integration. Who owns CS in your org — is that a separate team from sales?"
Already Have a Tool — ConnectWise / ScalePad▾
"We use ConnectWise." / "We have ScalePad / Ray Allen."
ConnectWise: PSA / ticketing only — complement it, don't replace it. "We integrate with ConnectWise via API so your service data flows into Planhat automatically. You keep your PSA, we add the CS and lifecycle layer on top."

ScalePad / Ray Allen: Asset lifecycle only — no CS capabilities. Planhat ingests their asset data and adds health scoring, renewal automation, and expansion playbooks.
Enterprise / Trust Concerns▾
"You're not big enough / we need an enterprise-grade solution."
Lead with logos: Lumen, Verizon, DXC, Telefónica, Comcast Business, Softcat, Telia Cygate. Reference security credentials (SOC 2, GDPR). Gartner named Planhat a Leader. "We work with companies like [relevant logo]. What specifically would you need to see from a security or enterprise readiness perspective?"
"We'd need to go through procurement / legal / InfoSec."
Good sign — they're engaged. Ask: "Who should I loop in from your side to make that process smooth? I can have our security documentation and SLA ready within 24 hours."
